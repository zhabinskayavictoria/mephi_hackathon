### Проблема 

Разработка новых лекарственных препаратов — сложный и длительный процесс, включающий синтез соединений и их биологическое тестирование. Современные методы машинного обучения позволяют ускорить этот процесс, предсказывая эффективность химических соединений до проведения лабораторных экспериментов.

В этом соревновании нам предстояло выступить в роли ML и Data-инженеров в команде фармацевтической разработки.
Химики предоставили данные о свойствах молекул и их биологической активности против вируса гриппа. Задача — построить модели, способные предсказывать эффективность новых соединений.

### Предсказываемые показатели

| Показатель | Описание |
|------------|----------|
| **IC50** (mM) | Концентрация, при которой вещество подавляет 50% активности вируса |
| **CC50** (mM) | Концентрация, при которой вещество токсично для 50% клеток |
| **SI** | Индекс селективности |

# Краткое описание решений
1. **[`Final_pipeline_team7.ipynb`](Final_pipeline_team7.ipynb)**

В этом ноутбуке используются отдельные модели `CatBoostRegressor` для предсказания `IC50` и `CC50`. Для `SI` применяется relation-aware подход: показатель рассчитывается преимущественно через связь `SI ≈ CC50 / IC50`, а затем дополнительно корректируется отдельной моделью `ExtraTreesRegressor`.

Финальное значение `SI` считается как взвешенная комбинация:

```text
SI_final = 0.88 × SI_formula + 0.12 × SI_model
```

где:

```text
SI_formula = CC50_pred / IC50_pred
```

Для стабилизации результата используется clipping предсказаний `IC50` по квантилям `0.003–0.997`, так как `IC50` находится в знаменателе при расчёте `SI` и экстремальные значения могут сильно ухудшать итоговую метрику.

2. **[`2rounds_ensemble_team7.ipynb`](2rounds_ensemble_team7.ipynb)**

Решение основано на ансамбле моделей и двух раундах pseudo-labeling.

Основные этапы:

- пропуски заполняются медианой через `SimpleImputer`;
- для каждого таргета отбираются топ-80 признаков по важности `RandomForestRegressor`;
- для каждого таргета обучаются три модели: `RandomForestRegressor`, `ExtraTreesRegressor` и `HistGradientBoostingRegressor`;
- для стабилизации обучения используются преобразования таргетов `log1p` и `sqrt`;
- предсказания моделей объединяются через оптимальные веса, подобранные методом `differential_evolution`;
- выполняются два раунда pseudo-labeling: предсказания на тесте добавляются в обучение с весом `0.5`.

В ноутбуке используется multi-seed кросс-валидация: предсказания усредняются по нескольким значениям seed, что снижает зависимость результата от одного конкретного разбиения на фолды.

3. **[`From_Descriptors_to_Ratio_Conditional_Forecasting_team7.ipynb`](From_Descriptors_to_Ratio_Conditional_Forecasting_team7.ipynb)**

Это исследовательский пайплайн, в котором основной акцент сделан на анализе молекулярных дескрипторов, создании дополнительных химически осмысленных признаков и использовании связи между таргетами.

В ноутбуке выполняются:

- анализ распределений `IC50`, `CC50` и `SI`;
- анализ пропусков и корреляций;
- создание новых признаков, например `density_surface`, `hydrophobicity_density`, `branching_index`, `compactness_ratio`, `cyclicity`, `flexibility_ratio`;
- удаление низковариативных и сильно скоррелированных признаков;
- обработка выбросов по квантилям или IQR;
- подбор моделей через `RandomizedSearchCV`.

Для `IC50` и `CC50` сравнивались `XGBoost`, `LightGBM` и `Ridge`. По результатам кросс-валидации лучшими стали:

| Таргет | Лучшая модель | Логарифмирование | CV RMSE |
|---|---|---|---:|
| `IC50` | `LightGBM` | Нет | 316.5577 ± 45.3213 |
| `CC50` | `XGBoost` | Нет | 448.2962 ± 69.3013 |

Для `SI` отдельная модель оказалась нестабильной и почти не объясняла дисперсию таргета. Поэтому финальное значение `SI` рассчитывается напрямую через предсказанные `IC50` и `CC50`:

```text
SI = CC50_pred / IC50_pred
```

4. **[`CatBoost_Per_Target_team7.ipynb`](CatBoost_Per_Target_team7.ipynb)**

В этом решении проверялась гипотеза, что для каждого таргета лучше обучать отдельную модель, а не использовать одну общую multi-target модель.

Основные шаги:

- нормализация названий колонок;
- удаление 18 константных признаков;
- использование 192 числовых признаков после очистки;
- 5-fold `KFold` с перемешиванием и фиксированным seed;
- обучение отдельного `CatBoostRegressor` для `IC50`, `CC50` и `SI`;
- подбор более консервативных параметров для `SI`, так как этот таргет имеет наибольшую вариативность;
- сохранение OOF-предсказаний и важности признаков.

Подход улучшил качество по сравнению с базовой CatBoost MultiRMSE-моделью, потому что каждая модель специализируется на одном биологическом показателе и не вынуждена искать компромисс между разными таргетами.

5. **[`ensemble_log_transform_team7.ipynb`](ensemble_log_transform_team7.ipynb)**

Это ансамблевое решение, в котором все три таргета обучаются в логарифмическом пространстве.

Для каждого таргета используется ансамбль из трёх моделей:

- `HistGradientBoostingRegressor`;
- `RandomForestRegressor`;
- `ExtraTreesRegressor`.

Основные особенности:

- целевые переменные преобразуются через `log1p`;
- для `RandomForestRegressor` и `ExtraTreesRegressor` пропуски заполняются через `KNNImputer`;
- для устойчивого масштабирования используется `RobustScaler`;
- качество оценивается на 5-fold `KFold`;
- финальное предсказание получается простым усреднением трёх моделей;
- после предсказания значения возвращаются в исходную шкалу через `expm1`.

Решение показало, что логарифмирование помогает стабилизировать обучение при широком диапазоне значений таргетов, однако прямое предсказание `SI` оказалось менее эффективным, чем использование связи `SI ≈ CC50 / IC50` в лучших решениях.

# Подробное описание лучшего решения 


Лучшим решением команды стал ноутбук **`Final_pipeline_team7.ipynb`**, который получил Kaggle-score **269.45188**. Финальный подход основан на сочетании классического машинного обучения и анализа связи между целевыми переменными.

### 1. Подготовка окружения

В начале ноутбука устанавливается библиотека `catboost`, импортируются необходимые библиотеки и фиксируется random seed.

Для воспроизводимости фиксируются:

- `random.seed`;
- `numpy.random.seed`;
- `PYTHONHASHSEED`;
- `random_seed` в CatBoost;
- `random_state` в ExtraTrees;
- `random_state` в KFold.

Это позволяет получать стабильные результаты при повторном запуске ноутбука.

### 2. Загрузка и первичная обработка данных

В решении используются три файла:

```text
train.csv
test.csv
sample_submission.csv
```

После загрузки данные имеют следующую структуру:

- `train.csv` — 751 строка и 214 колонки;
- `test.csv` — 250 строк и 211 колонок;
- `sample_submission.csv` — 250 строк и 4 колонки.

Далее выполняется нормализация названий колонок: из них удаляются пробелы, запятые и единицы измерения, например `IC50, mM` преобразуется в `IC50`.

### 3. Проверка качества данных

На этапе EDA проверяются:

- типы данных;
- пропуски;
- дубликаты;
- константные признаки;
- распределение целевых переменных.

Все признаки имеют числовой тип, поэтому дополнительное кодирование категориальных переменных не требуется. 

В данных были обнаружены признаки с пропущенными значениями. В каждом из них было всего по 2 пропуска в train. Для CatBoost отдельное заполнение пропусков не выполнялось, так как эта модель умеет работать с пропущенными значениями самостоятельно. Для ExtraTrees пропуски заполнялись медианой внутри `Pipeline`.

Также были удалены константные признаки, так как они не несут полезной информации для модели. После очистки осталось **192 признака**.

### 4. Валидация

Для оценки качества используется 5-fold кросс-валидация:

```python
KFold(
    n_splits=5,
    shuffle=True,
    random_state=SEED
)
```

Для каждого таргета считается отдельный RMSE, после чего берётся среднее значение.

### 5. Модели для IC50 и CC50

Для таргетов `IC50` и `CC50` обучаются отдельные модели **CatBoostRegressor**.

Используются регуляризованные параметры:

```python
iterations = 5000
learning_rate = 0.015
depth = 3
l2_leaf_reg = 10
early_stopping_rounds = 300
loss_function = "RMSE"
random_seed = 42
```

Такой подход позволяет обучать устойчивые модели и снижать риск переобучения на небольшой обучающей выборке. После предсказания значения дополнительно ограничиваются снизу нулём, так как концентрации и индекс селективности не могут быть отрицательными.

### 6. Модель для SI

Для `SI` дополнительно обучается модель **ExtraTreesRegressor**. Перед моделью используется `SimpleImputer` со стратегией `median`, чтобы заполнить пропущенные значения:

```python
Pipeline([
    ("imputer", SimpleImputer(strategy="median")),
    ("model", ExtraTreesRegressor(...))
])
```

Параметры ExtraTrees:

```python
n_estimators = 1200
max_depth = 12
min_samples_leaf = 2
max_features = 0.7
random_state = 42
n_jobs = -1
```

Для обучения `SI` используется логарифмическое преобразование таргета:

```python
log1p(SI)
```

После предсказания выполняется обратное преобразование:

```python
expm1(prediction)
```

Это помогает модели лучше работать с распределением `SI`, где встречаются большие значения.

### 7. Использование связи между таргетами

Ключевой частью финального решения стал анализ связи между целевыми переменными.

Показатель `SI` по смыслу связан с отношением:

```text
SI ≈ CC50 / IC50
```

Поэтому финальный пайплайн не рассматривает `SI` как полностью независимую переменную. Вместо этого используется relation-aware подход:

1. модель CatBoost предсказывает `IC50`;
2. модель CatBoost предсказывает `CC50`;
3. рассчитывается формульное значение:

```text
SI_formula = CC50_pred / IC50_pred
```

4. отдельно обученная модель ExtraTrees даёт дополнительный прогноз `SI_model`;
5. итоговый `SI` считается как blend:

```text
SI_final = 0.88 × SI_formula + 0.12 × SI_model
```

Главный прирост качества был получен именно за счёт такого использования связи между таргетами.

### 8. Стабилизация IC50 и SI

Так как `IC50` находится в знаменателе при расчёте `SI`, экстремально маленькие или большие значения `IC50` могут сильно исказить итоговый `SI`.

Чтобы уменьшить влияние выбросов, используется clipping `IC50` по квантилям обучающей выборки:

```python
ic50_clip_low = train["IC50"].quantile(0.003)
ic50_clip_high = train["IC50"].quantile(0.997)
```

После этого `SI_formula` также ограничивается сверху по квантилю `0.995` таргета `SI` в обучающей выборке.

Эта постобработка делает предсказания более стабильными и улучшает итоговую метрику.

### 9. Формирование submission

В него записываются:

- `IC50` — предсказания CatBoost после clipping;
- `CC50` — предсказания CatBoost;
- `SI` — relation-aware blend формулы и ExtraTrees-модели.

Итоговый файл сохраняется под названием:

```text
submission_strategy9_ic50_clip_0003_0997.csv
```

Именно этот подход дал лучший результат команды на Kaggle.


## Воспроизводимость во всех .ipynb

Во всех ноутбуках фиксируются значения random seed, чтобы результаты можно было воспроизвести при повторном запуске.

Основной seed проекта:

```python
SEED = 42
```

В разных ноутбуках он используется для:

- `numpy.random.seed(...)`;
- `random.seed(...)`;
- `PYTHONHASHSEED`;
- `random_state` в моделях sklearn;
- `random_seed` в CatBoost;
- `random_state` в `KFold`;
- `random_state` в `RandomizedSearchCV`;
- seed в процедуре подбора весов ансамбля.

В `2rounds_ensemble_team7.ipynb` дополнительно используется multi-seed подход:

```python
BASE_SEED = 42
SEEDS = [7, 13, 42, 123, 456]
```

Предсказания усредняются по нескольким seed, что делает результат менее зависимым от одного конкретного разбиения данных.

Во всех решениях также контролируется отсутствие утечек данных:

- тестовые таргеты не используются;
- внешние данные не применяются;
- трансформации признаков обучаются только на train-части внутри кросс-валидации или на обучающей выборке перед финальным предсказанием;
